## 项目架构
### 1. 技术栈
- 前端 ：Vue 3 + Composition API + Pinia + Vue Router + ECharts
- 后端 ：Spring Boot 3.2.2 + JPA + MySQL
- 认证 ：基于Token的认证机制
- 文件存储 ：阿里云OSS
### 2. 系统架构
项目采用前后端分离架构：

- 前端 ：单页应用，使用Vue 3构建，包含学生、教师和管理员三个角色的界面
- 后端 ：RESTful API，使用Spring Boot构建，提供数据服务和业务逻辑处理
### 3. 核心功能模块 3.1 用户管理
- 支持三种角色：学生、教师、管理员
- 包含用户注册、登录、个人信息管理
- 教师认证机制（验证码验证） 3.2 课程管理
- 教师可以创建、编辑、删除课程
- 支持通过邀请码添加学生
- 课程分类管理（必修、选修、公共课、其他）
- 课程统计功能 3.3 资料管理
- 支持上传、下载、分享资料
- 资料分类和统计
- 资料互动（点赞、收藏、评论） 3.4 作业管理
- 教师发布作业
- 学生提交作业
- 作业状态管理（未开始、进行中、已完成）
- 作业优先级设置 3.5 测验管理
- 支持创建练习测验和正式考试
- 题目管理
- 测验状态管理（草稿、已发布、已关闭）
- 测验成绩统计和分析 3.6 学习统计
- 学习时长统计
- 测验成绩分析
- 学习趋势图表展示
### 4. 数据模型 4.1 核心实体
- User ：用户信息，包含角色、个人信息等
- Course ：课程信息，包含课程基本信息、邀请码等
- Task ：作业信息，包含任务详情、状态等
- Material ：资料信息，包含文件信息、统计数据等
- Quiz ：测验信息，包含测验详情、题目等
- Question ：题目信息
- QuizRecord ：测验记录
- StudentCourse ：学生课程关联
- StudyRecord ：学习记录 4.2 关系设计
- 用户与课程：一对多（教师创建课程）
- 学生与课程：多对多（通过StudentCourse表）
- 课程与资料/作业/测验：一对多
- 测验与题目：一对多
- 用户与测验记录：一对多
### 5. 界面设计 5.1 布局结构
- 采用左右布局：左侧导航栏，右侧内容区
- 响应式设计，适配不同屏幕尺寸
- 统一的设计风格和组件库 5.2 主要页面
- 学生端 ：课程学习、资料中心、学习统计
- 教师端 ：仪表盘、课程管理、作业管理、测验管理、资料管理
- 管理员端 ：仪表盘、用户管理、内容管理、邀请码管理
### 6. 技术亮点
1. 模块化设计 ：前端采用组件化开发，后端采用分层架构
2. 响应式布局 ：适配不同设备屏幕
3. 数据可视化 ：使用ECharts实现学习统计图表
4. 邀请码机制 ：课程邀请码生成和验证
5. 权限管理 ：基于角色的权限控制
6. 文件上传 ：集成阿里云OSS存储
7. 实时统计 ：学习数据实时统计和分析
### 7. 项目特色
1. 多角色系统 ：满足学生、教师、管理员不同需求
2. 学习辅助功能 ：提供作业、测验、资料等全面的学习辅助工具

## 总结
这是一个功能完整、架构清晰的学习辅助系统，采用现代技术栈构建，满足了学生、教师和管理员的不同需求。系统具有良好的可扩展性和可维护性，为学习管理提供了全面的解决方案。

项目的核心价值在于：

- 为教师提供了便捷的课程管理工具
- 为学生提供了丰富的学习资源和辅助工具
- 为管理员提供了有效的系统管理功能
- 通过数据统计和分析，帮助用户了解学习情况，提升学习效率
该系统不仅实现了基本的学习管理功能，还通过数据可视化等技术手段，为用户提供了更加直观、高效的学习体验。
